  Ensemble Model Architecture Weighted Soft Voting

The final prediction is computed as a weighted average of probability scores from three classifiers — each with complementary strengths for catching different fraud patterns.

LinearSVC 60% weight
Support Vector Machine with linear kernel. Best at finding the maximum-margin hyperplane separating fraud from legit messages. Strongest performer in high-dimensional TF-IDF feature spaces. Uses Platt scaling for probability calibration.
Logistic Regression 20% weight
Provides well-calibrated probabilities and interpretable feature coefficients. Powers the Word Risk Heatmap — each word's coefficient directly maps to its visual risk colour in the explainability feature.
MultinomialNB 20% weight
Multinomial Naive Bayes — fast and effective for text classification. Captures word-frequency distributions well and provides a complementary probabilistic view, especially effective for phishing URL patterns.

  Novel Features

Word Risk Heatmap

Token-level visual explainability using Logistic Regression coefficient weights. Every word in the original message is highlighted by its fraud signal strength — RED (high), AMBER (medium), GREEN (low). Non-technical fraud victims can immediately see what triggered the alert. No other student project implements this approach.

Token Scoring LR Coefficients Hover Tooltips

Fraud Velocity Tracker

Cross-complaint repeat offender detection. Every phone number and URL in a FRAUD/SUSPICIOUS message is recorded in a persistent JSON threat intelligence database. Repeat offenders are escalated: LOW → MEDIUM → HIGH → CRITICAL. Modelled after how TRAI builds telecom fraud blacklists from crowd-sourced data.
